WPAIPublisher AI → WordPress

Codex / ChatGPT / Claude Code / Copilot の出力を、品質ゲート・Git CI/CD・RAG・自律エージェント経由で WordPress へ継続反映する再現可能なワークフローです。

Python · CLI Claude Code WP-CLI · REST Playwright Git · Actions RAG · TF-IDF
Architecture エンドツーエンド・パイプライン

AI 出力を intake/ で受け取り、マルチ AI 変換 → 品質ゲート → ビジュアル回帰 → Git PR → ステージング → 本番までを一連で実行します。

  • マルチ AI — ChatGPT · Codex · Claude Code · Copilot
  • 品質ゲート — HTML · SEO · a11y · セキュリティ · パフォーマンス
  • Git CI/CD — PR · staging · main · ロールバック
  • Playwright — デスクトップ / モバイル画面比較
  • RAG — 過去変換パターンのベクトル再利用
  • Agent — workflow/pipeline.yaml による自律実行
Agent 自律実行（推奨フロー）

1 コマンドで intake → RAG → 変換準備 → 品質ゲート → PR → ステージングまで進行。手動介入が必要なステージでは一時停止します。

  • python wpaipublish.py agent run <session> — 開始
  • agent resume — Claude Code 変換完了後に再開
  • agent resume --approve — 本番デプロイ承認
  • 中断: wp_conversion · deploy_review · deploy_production

↓ セットアップ・変換・品質・デプロイの手順は下へ

  1. 1. 初回セットアップ

    依存関係と設定ファイルを用意します。API キーは config/.env のみに保存してください。

    • pip install -r requirements.txt
    • npm install && npx playwright install chromium
    • cp config/*.example.yaml config/（拡張子を調整）
    • cp config/.env.example config/.env を編集
    • 必須: WP_STAGING_* / 任意: OPENAI_API_KEY · ANTHROPIC_API_KEY
  2. 2. ナレッジベース構築（RAG）

    プロンプト・ドキュメント・過去セッションをインデックスし、変換時に再利用します。

    • python wpaipublish.py knowledge index --rebuild
    • セッション追加: knowledge index --session <id>
    • 検索: knowledge retrieve --session <id>
    • 出力: output/<id>/rag_context.md
  3. 3. AI 出力の投入（intake）

    Codex 等の成果物を所定形式で受け取ります。manifest.json は必須です。

    • 配置: intake/incoming/<package>/
    • 参考: intake/example/（hero-section サンプル）
    • 整形プロンプト: prompts/prepare-intake.md
    • 検証: python wpaipublish.py intake validate intake/incoming/<package>
    • target.type: block · theme · page · template-part · custom-css
  4. 3b. 複数 HTML から処理対象を選択

    1 フォルダに複数の HTML がある場合、処理したいファイルだけを選んで intake → validate → convert prepare まで実行できます。 同名の CSS/JS と、HTML 内の相対参照アセットは自動で同梱されます。

    • 一覧: python wpaipublish.py intake list <folder>
    • 対話選択: python wpaipublish.py intake select <folder> --interactive
    • 指定してパイプライン開始:
      python wpaipublish.py intake pipeline <folder> --select hero.html --select pages/about.html --target-type page
    • Web UI: ナビの「HTML選択」(/pipeline) → フォルダパス入力 → チェック選択 → 「パイプライン開始」
    • サンプル: intake/samples/multi-html/（hero / about / contact）
    • 生成物: intake/incoming/selected-* と output/<session>/
    • 続き: ai route → convert mark-done → quality run → deploy staging
  5. 4. 変換セッション作成

    Claude Code 向けの入力パッケージと指示ファイルを生成します。

    • python wpaipublish.py convert prepare intake/incoming/<package> --session-id my-session
    • 生成: output/my-session/source/ · CLAUDE_INSTRUCTIONS.md
    • マルチ AI: python wpaipublish.py ai route --stage wp_conversion --session my-session
    • プロバイダー確認: python wpaipublish.py ai list
  6. 5. WordPress 向け変換（Claude Code）

    prompts/convert-to-wp.md に従い、HTML/CSS/JS をブロック・テーマ等へ変換します。

    • 出力先: output/<session>/wordpress/
    • 自己検証: prompts/validate-theme.md
    • 完了マーク: python wpaipublish.py convert mark-done <session>
    • 参考実装: docs/examples/hero-block/
  7. 6. 品質ゲート

    HTML / SEO / アクセシビリティ / セキュリティ / パフォーマンスを自動検査します。blocking 失敗時はデプロイ不可です。

    • python wpaipublish.py quality run <session> --stage pr_merge
    • 基本検証: validate run <session>（PHP 構文等）
    • 設定: config/quality-gates.yaml
    • 結果: output/<session>/quality_gates.json
  8. 7. ビジュアル回帰テスト

    Playwright でデスクトップ / モバイルの画面を比較します。初回はベースライン更新が必要です。

    • 初回: python wpaipublish.py visual run <session> --update
    • 以降: visual run <session> --env staging
    • 保存先: visual/baseline/ · current/ · diff/
  9. 8. Git PR & CI/CD

    feature ブランチから staging へ PR。GitHub Actions が品質ゲートとビジュアルテストを実行します。

    • python wpaipublish.py git pr <session>
    • ブランチ: wpai/<session> → base: staging
    • CI: .github/workflows/wpaipublisher-ci.yml
    • Secrets: WP_STAGING_* · WP_PROD_*
  10. 9. ステージングデプロイ & 確認

    必ずステージングで表示・動作を確認してから本番へ進みます。

    • python wpaipublish.py deploy staging <session>
    • Git 経由: deploy staging <session> --via-git
    • 確認: レスポンシブ · JS · 既存ページ影響 · コンソールエラー
    • レビュー: prompts/deploy-review.md
  11. 10. 本番デプロイ

    --confirm 必須。デプロイ前に自動バックアップが作成されます。

    • python wpaipublish.py deploy production <session> --confirm
    • Git: git deploy production <session> --confirm
    • Agent: agent resume <session> --approve
    • 履歴: deployments/history.jsonl
  12. 11. ロールバック

    障害時はバックアップまたは Git revert で復旧します。詳細は docs/ROLLBACK.md。

    • ファイル: python wpaipublish.py rollback <session> production --confirm
    • Git: git rollback <session> production --confirm
    • バックアップ: deployments/backup-prod-*
    • キャッシュ: wp cache flush
  13. 12. 運用ルール（必須）

    再現性と安全性のための固定ルールです。

    • 本番への直接デプロイ禁止（ステージング必須）
    • manifest.json なしの intake は受け付けない
    • 品質ゲート / ビジュアルテストをスキップしない
    • 本番は --confirm または Agent --approve 必須
    • 処理済み intake は intake/processed/ へ移動
    • 状態確認: python wpaipublish.py status
